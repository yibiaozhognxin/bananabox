# bananabox
Bananabox 是一个基于 Node.js 的图像生成工作台，支持提示词生成、参考图上传、比例与分辨率选择、历史记录管理、图片预览、图片编辑和管理员后台配置。它适合私有部署、个人创作和小团队内部使用。

## 功能介绍

### 图片生成
- 支持输入提示词生成图片
- 支持上传多张参考图
- 支持选择生成比例
- 支持选择分辨率
- 支持生成任务排队（每个ID最多5并行任务）

### 历史管理
- 自动保存生成结果到本地
- 支持预览大图
- 支持下载图片
- 支持删除历史记录和本地图片文件

### 图片编辑
- 支持对已有图片或这自主上传图片进行编辑生成
- 支持不同颜色画笔区域填写不同修改指令

### 管理员后台
- 支持配置上游 API 地址
- 支持配置 API Key
- 支持配置不同分辨率对应的模型 ID
- 支持开启或关闭 1K / 2K / 4K 分辨率
- 支持查看用户列表
- 支持禁用用户
- 支持删除用户及其本地数据
- 支持新用户注册开关

## 推荐使用场景

### 适合场景
- 个人私有图像生成工作台
- 小团队内部创作工具
- 需要保留历史记录的图片生成环境

### 不太适合的场景
- 完全公开、无需配置即可使用的公共服务
- 权限要求很高的大型平台
- 需要复杂多租户系统的企业级场景

## 环境要求

- Node.js 18+
- 可用的上游图像生成接口
- 具有写入权限的本地磁盘
- Windows / Linux / macOS 均可运行

## 部署教程

### 1. 获取项目
将整个项目目录上传到服务器或克隆到本地。

### 2. 安装依赖
在项目根目录执行：

```bash
npm install
```

### 3. 启动服务
执行：

```bash
npm start
```

默认监听端口为 `3000`。 第一个注册的用户ID为管理员

### 4. 打开页面
浏览器访问：

```text
http://127.0.0.1:3000/nanobanana.html
```

### 5. 服务器部署建议
- 建议使用 `pm2`、`systemd` 或其他守护进程工具
- 如果想用 80 端口，可以使用 Nginx 反向代理到 `3000`
- 建议定期备份 `data/` 目录
- 图片和历史数据会保存在本地，请确保磁盘空间足够


## 设置说明

### API URL
- 用途：上游图像生成接口地址
- 示例：`https://example.com`

### API Key
- 用途：访问上游接口的鉴权密钥

### 分辨率模型映射
当前支持：
- 1K
- 2K
- 4K

每个分辨率都可以单独配置：
- 是否开启
- 对应模型 ID


### 新用户注册开关
- 开启后允许新用户进入系统
- 关闭后只允许已有用户继续使用


## 本地数据说明

### 1. `data/system.json`
保存系统配置：
- 管理员 ID
- 是否允许新用户注册
- API 地址
- API Key
- 分辨率模型映射

### 2. `data/users.json`
保存用户元数据：
- userId
- role
- status
- createdAt
- lastSeenAt

### 3. `data/history.json`
保存历史任务记录：
- prompt
- model
- ratio
- 生成结果路径
- 缩略图路径
- 时间戳
- 状态

### 4. `data/users/<userId>/images/`
保存生成的原图文件

### 5. `data/users/<userId>/thumb/`
保存缩略图

## 常见问题

### 1. 页面打不开
- 检查 `npm start` 是否启动成功
- 检查端口 `3000` 是否被占用
- 检查服务器防火墙是否放行端口

### 2. 无法生成图片
- 检查 API URL 是否正确
- 检查 API Key 是否有效
- 检查分辨率模型 ID 是否填写正确
- 检查上游接口是否可访问
